Introdução

Primeiros passos com o Payment Gateway Microservice Asaas
Esta API permite a integração com o serviço Asaas para gerenciamento de clientes e pagamentos, oferecendo endpoints seguros e flexíveis para facilitar a automação financeira do seu negócio.

Autenticação

Todas as requisições devem incluir o header access_token com a chave de acesso fornecida pelo Asaas.

  • O token deve ser enviado em todas as chamadas à API.
  • Tokens possuem permissões específicas, verifique com o administrador do sistema.

Ambientes

Utilize o parâmetro de query ?env=sandbox ou ?env=prod para selecionar o ambiente desejado.

  • Sandbox: Ambiente de testes, sem impacto financeiro real.
  • Produção: Ambiente real, utilize com cautela.

Principais Recursos

  • Cadastro e gerenciamento de clientes
  • Emissão e acompanhamento de cobranças
  • Recebimento de notificações automáticas
  • Consulta de status de pagamentos
  • Relatórios e exportação de dados

Exemplo de Requisição

GET /api/customers?env=sandbox
Headers:
  access_token: SUA_CHAVE_DE_ACESSO

Boas Práticas

  • Mantenha seu access_token seguro e nunca compartilhe publicamente.
  • Utilize sempre HTTPS para comunicação com a API.
  • Implemente tratamento de erros para respostas inesperadas.
