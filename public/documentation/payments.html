Pagamentos

Gerencie cobranças e pagamentos de clientes.

Criar Cobrança

Método: POST /payments
Cria uma nova cobrança para um cliente.

Body (JSON)

Campo Tipo Descrição Obrigatório
customerId string ID do cliente que irá receber a cobrança Sim
value number Valor da cobrança individual de cada parcela (caso totalValue não seja informado) ou valor base para cálculo das parcelas Sim
dueDate string Data de vencimento da cobrança ou da primeira parcela (YYYY-MM-DD) Sim
billingType string Tipo de cobrança: BOLETO, CREDIT_CARD ou PIX Sim
totalValue number Valor total da cobrança. Se preenchido, o valor de cada parcela será calculado automaticamente considerando value e totalValue. Não

Headers

  • access_token: Chave de acesso do Asaas (obrigatório)

Parâmetros de Query

  • environment (opcional): SANDBOX ou PROD

Exemplo de Requisição

POST /payments?environment=SANDBOX
Headers:
  access_token: SUA_CHAVE_DE_ACESSO
Body:
{
  "customerId": "cus_123456789",
  "billingType": "BOLETO",
  "value": 100.00,
  "dueDate": "2025-05-10"
}

Exemplo de Resposta

{
  "id": "pay_987654321",
  "status": "PENDING",
  "dueDate": "2025-05-10",
  "value": 100.00
}

Listar Parcelas de um Plano

Método: GET /payments/installments/:id
Lista todas as parcelas de um plano de pagamento.

Headers

  • access_token: Chave de acesso do Asaas (obrigatório)

Parâmetros de Query

  • environment (opcional): SANDBOX ou PROD

Exemplo de Requisição

GET /payments/installments/inst_123456789?environment=SANDBOX
Headers:
  access_token: SUA_CHAVE_DE_ACESSO

Exemplo de Resposta

{
  "data": [
    {
      "id": "pay_987654321",
      "status": "PENDING",
      "dueDate": "2025-05-10",
      "value": 100.00
    }
  ],
  "totalCount": 1
}

Campos de Resposta

Campo Tipo Descrição
id string ID único da cobrança
status string Status da cobrança (PENDING, RECEIVED, etc.)
dueDate string Data de vencimento
value number Valor da cobrança

Observações

  • O header access_token é obrigatório em todas as rotas de pagamentos.
  • O parâmetro environment define se a operação será feita em SANDBOX (teste) ou PROD (produção). Se não informado, o padrão é SANDBOX.
  • Todos os campos obrigatórios devem ser preenchidos para criar uma cobrança.
