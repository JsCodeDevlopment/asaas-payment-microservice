Notificações

Gerencie notificações de clientes, incluindo consulta e atualização de preferências de envio.

Endpoints

  • GET /notifications/customer/:id — Lista notificações de um cliente
  • PUT /notifications/:id — Atualiza uma notificação
  • PUT /notifications/batch — Atualiza notificações em lote

Parâmetros Comuns

Header Descrição Obrigatório
access_token API Key do Asaas para autenticação Sim
Query Descrição Obrigatório
environment Escolhe o ambiente Asaas (SANDBOX ou PROD) Não (padrão: SANDBOX)

Exemplo de Resposta

{
  "id": "noti_123",
  "customer": "cus_456",
  "event": "PAYMENT_RECEIVED",
  "media": "EMAIL",
  "enabled": true
}

Campos de Notificação

Campo Tipo Descrição
id string Identificador da notificação
customer string ID do cliente
event string Evento relacionado (ex: PAYMENT_RECEIVED)
media string Meio de envio (EMAIL, SMS, WHATSAPP, VOICE)
enabled boolean Se a notificação está ativa

Exemplo de Atualização

{
  "media": "EMAIL",
  "enabled": false
}

Observações

  • Os campos media aceitam apenas: EMAIL, SMS, WHATSAPP, VOICE.
  • O campo enabled define se a notificação será enviada ou não.
  • Para atualizar várias notificações, envie um array de objetos no endpoint /notifications/batch.
